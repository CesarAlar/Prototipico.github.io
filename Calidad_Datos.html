Validación de datos en un formulario


A continuación, podrás observar la imagen de un formulario en el cual podrás ingresar dos tipos diferentes de información. En el primer campo de entrada se solicita ingresar el nombre de un producto, mientras que en el segundo campo de entrada, de tipo numérico, se solicita ingresar la cantidad o precio del producto previamente mencionado. Esto se hace para extraer dicha información y generar nuevos precios. A los precios obtenidos se les aplicará la inflación correspondiente a cada mes de este año, generando así los nuevos precios mencionados anteriormente.





Al final de estos campos, se puede observar la implementación de un campo de entrada de tipo "submit". Al hacer clic en este campo, se ejecutará una función encargada de validar los datos ingresados por el usuario. Más adelante, se explicará en detalle la función que se ejecuta y cómo se validan los datos.

La siguiente imagen muestra el código utilizado para validar los datos que hemos introducido. Comenzaremos creando una función que se ejecutará al hacer clic en el botón del formulario. A continuación, se ejecutará la segunda línea de código, que se utiliza para prevenir el comportamiento predeterminado de un evento.





Después, haremos uso del objeto global "campos" y luego crearemos un objeto adicional llamado "precios". En el objeto "campos" se almacenarán los nombres de los productos ingresados, mientras que en el objeto "precios" se guardarán los precios correspondientes a dichos productos.


Al asignar los nombres de los productos al objeto "campos", aplicaremos la función "trim()" para eliminar cualquier espacio en blanco adicional que puedan tener. Esta función es útil para eliminar los espacios sobrantes.


En el segundo objeto, le cambiaremos el tipo de dato a los precios de los productos y pasaran a ser de tipo número.





En esta línea de código, se extraen los nombres de los productos. Sin embargo, estos datos se someten a una sencilla limpieza utilizando la función "trim()" mencionada anteriormente.

También se puede observar que se declara una estructura condicional "if" que compara el contenido de las variables "produ1" y "produ2" con comillas simples, representando un string vacío.

El resto del código se encarga de verificar que los precios de los productos sean menores o iguales a cero. En caso de que se cumpla esta condición y los nombres de los productos sean igual a un string vacío, se procede a crear un mensaje para el usuario.





Antes de crear el mensaje, nos aseguramos de que exista la clase "alert". Esto se hace para evitar la creación de múltiples mensajes de advertencia para el usuario. Si la clase no existe, se crea el mensaje; de lo contrario, no se crea.


Para crear el mensaje, primero declaramos una variable que representará una etiqueta HTML, específicamente la etiqueta de párrafo. Luego, introducimos el mensaje del usuario en la etiqueta y le asignamos dos clases: "alerta" y "aler". La clase "alerta" se utiliza para personalizarlo con CSS, y luego agregamos el mensaje al documento HTML.


Usamos setTimeout para indicar que el mensaje se mostrará durante solo 3 segundos y luego se eliminará completamente del documento HTML.
